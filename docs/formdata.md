# .formdata.json

Stores the user-entered form data for a template. Written by the builder UI's form preview and read by the Liquid rendering middleware to render the template.

## Format

A JSON object whose keys match the field/container keys defined in `schema.json`:

```json
{
  "<field_key>": "<value>",
  "<container_key>": {
    "<nested_field>": "<value>"
  },
  "<repeatable_container>": [
    { "<field>": "<value>" },
    { "<field>": "<value>" }
  ]
}
```

## How It's Created

### Via the Builder UI

1. Run `template-builder dev <name>`
2. Open `http://localhost:4001/template-builder/`
3. Fill out the form preview on the right panel
4. Data auto-saves (300ms debounce) via `POST /template-builder/formdata`

The builder collects values from all `[name]` elements in the form preview and nests them according to dotted-path names (e.g. `contact.email` becomes `{ "contact": { "email": "..." } }`, `skills[0].name` becomes `{ "skills": [{ "name": "..." }] }`).

### Manually

Create `.formdata.json` in your template directory with any text editor. The structure must mirror your schema's field keys.

## Mapping to Liquid Template Variables

The entire `.formdata.json` is loaded as the Liquid rendering context. Field keys become directly accessible as Liquid variables:

| .formdata.json | Liquid Template Syntax |
|---|---|
| `{ "title": "Hello" }` | `{{ title }}` |
| `{ "contact": { "email": "a@b.com" } }` | `{{ contact.email }}` |
| `{ "items": [{ "name": "A" }, { "name": "B" }] }` | `{% for item in items %}{{ item.name }}{% endfor %}` |
| `{ "photo": "/uploads/uuid.jpg" }` | `<img src="{{ photo }}">` |
| `{ "bio": "Hello\nWorld" }` | `{{ bio | newline_to_br }}` |

## Naming Convention

- **Simple fields**: flat key matching the schema field key
- **Container fields**: nested object keyed by schema container key, then by child field keys
- **Repeatable containers**: array of objects (never a flat array), indexed numerically (auto-generated by builder). Each object's keys are the child field keys from the schema — even if the group has only one child field, items are still objects, not plain values.
- **File uploads**: stored as the server upload path string (e.g. `/uploads/uuid-filename.pdf`)

## Examples

### Simple fields only

```json
{
  "template_title": "My Portfolio",
  "template_description": "Welcome to my personal website built with template-builder."
}
```

Renders: `{{ template_title }}` → "My Portfolio"

### With a non-repeatable container

```json
{
  "template_title": "Contact Page",
  "contact": {
    "email": "user@example.com",
    "phone": "+1-555-0123",
    "address": "123 Main St"
  }
}
```

Renders: `{{ contact.email }}` → "user@example.com"

### With a repeatable container (work experience)

```json
{
  "template_title": "Resume",
  "experiences": [
    {
      "company": "Acme Corp",
      "role": "Engineer",
      "start_date": "2020-01",
      "end_date": "2023-12",
      "description": "Built stuff"
    },
    {
      "company": "Startup Inc",
      "role": "Senior Engineer",
      "start_date": "2024-01",
      "end_date": "",
      "description": "Leading team"
    }
  ]
}
```

Template usage:
```liquid
{% for exp in experiences %}
  <div class="entry">
    <h3>{{ exp.role }} at {{ exp.company }}</h3>
    <p>{{ exp.start_date }} — {{ exp.end_date }}</p>
    <p>{{ exp.description }}</p>
  </div>
{% endfor %}
```

### With a single-field repeatable container (skills)

Even with only one child field, the repeatable container produces an array of objects, not a flat array:

```json
{
  "template_title": "Resume",
  "skills": [
    { "skill": "JavaScript" },
    { "skill": "Python" },
    { "skill": "TypeScript" }
  ]
}
```

Template usage — access the field key on each item:
```liquid
{% for skill in skills %}
  <li>{{ skill.skill }}</li>
{% endfor %}
```

This always works the same way regardless of how many fields the container has.

### With file uploads

```json
{
  "avatar": "/uploads/a1b2c3d4-portrait.jpg",
  "resume": "/uploads/e5f6g7h8-resume.pdf"
}
```

Template usage:
```liquid
<img src="{{ avatar }}" alt="Avatar">
<a href="{{ resume }}">Download Resume</a>
```

### With view fields (img)

```json
{
  "hero_image": "https://example.com/banner.jpg",
  "section_intro": "Welcome to my site"
}
```

Template usage:
```liquid
<img src="{{ hero_image }}" alt="Hero">
<p>{{ section_intro }}</p>
```

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/template-builder/formdata` | Save formdata (accepts any JSON body) |
| `GET` | `/template-builder/formdata` | Retrieve saved formdata (404 if none) |

## Tips

- `.formdata.json` is auto-created the first time you save from the builder UI
- Delete `.formdata.json` to reset all form data
- File/image fields store server-side file paths (uploaded via `/template-builder/upload`)
- For images, you can use either a URL string or a base64 data URI as the `content` value
- The builder UI's `collectFormData()` automatically handles dotted-path nesting and array indexing for repeatable containers
